Главная

Перед вами небольшая статья-шпаргалка, которая включает в себя команды удобные, но не всем знакомые команды Git, которые могут стать вашими постоянными спутниками в разработке.

git init - инициализирует репозиторий

git clone - эта команда клонирует репозиторий с GitHub себе на компьютер

Пример команды:

git clone ссылка с GitHub

Git config - функция, с помощью которой Вы привязываете свой аккаунт к GitHub.

Примеры команд:

git config --local user.name "Name"

git config --local user.email "your_email@example.com"

git remote add - команда, которой устанавливается подключение к удаленному серверу и git репозиторию на нем.

Пример: git remote add https://github.com/MrKrishnaAgarwal/Git-CheatSheet.git

git remote -v - эта команда используется для просмотра подключенных удаленных репозиториев.

git status - эта команда используется для просмотра статуса файлов в вашем локальном репозитории. Отслеживаются ли файлы? Не отслеживается? Изменены ли они?

git add - эта команда переносит все старые и измененные файлы в раздел добавленных файлов

Пример:

git add index.html - для конкретного файла

git add style.css - для конкретного файла

git add -A - добавление (индексация) всех файлов: новых, измененных и удаленных.

git reset

Эта команда используется для отмены изменений, позволяет сбросить состояние проекта до нужного коммита.

Например:

Восстановить самый свежий коммит: git reset --hard HEAD^

Отмена отправленного коммита (возврат к состояние коммита 7880ae2): git reset --hard f7880ae2

git push - эта команда вносит изменения на удаленный репозиторий.

git push -f - удаляет из ветки на сервере все коммиты, которых нет в локальной версии, и записывает новые.

git commit - это команда Git для записи индексированных изменений в репозиторий.

git commit -m “first commit” - так мы покажем Git, что файл изменен. Изменения файла сохранятся в репозитории.

Команда git commit --amend вносит в предыдущий коммит изменения, которые подготовлены к коммиту. Используется и для редактирования предыдущего commit message, если в нём допущены ошибки.

git push - эта команда используется для отправки закоммиченных файлов в удаленный репозиторий (также известный как GitHub) в указанной ветке. Используйте эту команду, когда вы впервые отправляете файлы в удаленный репозиторий. Он зафиксирует место, куда вы отправляете эти файлы. И в следующий раз можно будет использовать команду git push.

git push -u

Ключ -u (полный вариант --set-upstream) создаёт в удалённом репозитории ветку, соответствующую локальной и связывает их.

git fetch - эта команда используется для скачивания изменений из удаленного или локального репозитория в свой локальный репозиторий.

git pull - эта команда используется для извлечения и загрузки содержимого из удаленного репозитория и немедленного обновления локального репозитория этим содержимым.

git branch - эта команда используется для создания ветки.

git branch -a - эта команда используется для предварительного просмотра всех веток в удаленном репозитории.

git branch -r - эта команда используется для предварительного просмотра всех веток на сервере.

git checkout - эта команда используется для переключения на ветки, которые вы уже посещали ранее.

Пример: git checkout Название ветки

git merge - эта команда используется для объединения веток. Для этого укажите ветку, которую вы хотите унаследовать изменения. И имя ветки, которое вы будете использовать вместе с этой командой, — это ветка, которая предоставит изменения.

git merge — abort - эта команда используется для отмены слияния.

Если нет ошибок, слияния будут успешными. Следовательно, это прерывание можно использовать только в ситуациях, когда слияние не удалось. Как понять, что нужно использовать эту команду? Ваш терминал скажет, что слияние не удалось. Он также может предложить вам исправить конфликты слияния.

Вот ещё один признак: ~/NextCloud/Documents/Web Projects/Cloud4Y (master)

Посмотрите в самый конец строки. В скобках написано (master). Это потому, что мы находимся в основной ветке. Если вы находитесь в ветке разработки, это будет означать (develop). Если смерджить не получилось, появится надпись (master|merging) или что-то в этом роде.

git reset -hard HEAD - эта команда удалит все изменения, внесенные вами в ваш локальный репозиторий, и обновит его до последней версии, которая была закоммичена на GitHub.

git diff - эта команда используется сравнения изменений.

git rebase - один из способов в git, позволяющий объединить изменения двух веток. У этого способа есть преимущество перед merge (слияние) — он позволяет переписать историю ветки, придав тот истории тот вид, который нам нужен.

git rebase -i HEAD~N - используется, когда нужно изменить описание коммита, сделанного задолго до последнего. Эта команда возвращает вас к n-му коммиту, и вы можете вносить в него изменения, вводя edit. Затем нужно сохранить изменения с помощью команды git push -f.

git stash - команда временного скрытия текущих изменений в репозитории Git.

git stash pop - это команда, которая удаляет изменения из stash и повторно применяет их к рабочей копии.
